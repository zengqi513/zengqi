## 优化内容

### 1. 服务保活机制

#### 前台服务增强
- **通知渠道**: 创建专用低重要性渠道，不显示角标
- **点击行为**: 添加 PendingIntent，点击通知打开应用
- **前台服务类型**: Android 10+ 使用 `FOREGROUND_SERVICE_TYPE_DATA_SYNC`
- **保活标志**: `setOngoing(true)` + `setCategory(CATEGORY_SERVICE)`

#### 心跳机制
- 30秒间隔心跳检查
- 自动更新 `lastPingTime`
- 独立协程运行，不阻塞主流程

### 2. 崩溃自动恢复

#### PaymentNotificationListener 增强
| 方法 | 功能 |
|------|------|
| `onCreate()` | 记录服务实例 |
| `onDestroy()` | 清理实例，延迟3秒后请求重新绑定 |
| `onListenerDisconnected()` | 断开后自动重新绑定 |
| `requestRebind()` | 静态方法，外部可调用重新绑定 |

#### ServiceWatchdog 看门狗
- **定时检查**: 每分钟检查一次服务健康状态
- **健康标准**: 2分钟内有活动且连接正常
- **自动恢复**: 异常时调用 `requestRebind()`
- **开机启动**: 设备重启后自动启动监控
- **精确唤醒**: 使用 `setExactAndAllowWhileIdle` 保证检查执行

### 3. 权限配置 (AndroidManifest.xml)

新增权限:
```xml
<uses-permission android:name="android.permission.FOREGROUND_SERVICE_DATA_SYNC" />
<uses-permission android:name="android.permission.WAKE_LOCK" />
<uses-permission android:name="android.permission.REQUEST_IGNORE_BATTERY_OPTIMIZATIONS" />
```

新增组件:
```xml
<receiver android:name=".service.ServiceWatchdog" ...>
    <intent-filter>
        <action android:name="android.intent.action.BOOT_COMPLETED" />
        <action android:name="com.autobookkeeper.action.CHECK_SERVICE" />
    </intent-filter>
</receiver>
```

### 4. 应用启动集成

MainActivity.onCreate() 中自动启动看门狗:
```kotlin
ServiceWatchdog.start(this)
```

## 服务状态监控

应用可通过以下方式检查服务状态:

```kotlin
// 检查是否连接
val isConnected = PaymentNotificationListener.isConnected

// 检查健康状态（2分钟内有活动）
val isHealthy = PaymentNotificationListener.isServiceHealthy()

// 手动请求重新绑定
PaymentNotificationListener.requestRebind(context)
```
